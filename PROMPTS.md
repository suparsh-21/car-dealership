# AI Prompts Used During Development

This file documents the major prompts used while developing the AutoVault Car Dealership Inventory Management System.

---

## Prompt 1 – Project Architecture

Design a full-stack Car Dealership Inventory Management System using React, Node.js, Express, and MongoDB. The application should support JWT authentication, role-based access control, vehicle CRUD operations, inventory management, search functionality, and follow REST API best practices.

---

## Prompt 2 – Authentication

Help implement secure JWT authentication with user registration, login, password hashing using bcryptjs, protected routes, and role-based authorization for Admin and Customer users.

---

## Prompt 3 – Vehicle CRUD

Generate REST API endpoints for adding, updating, deleting, and retrieving vehicles. Only administrators should be able to modify inventory.

---

## Prompt 4 – Search Feature

Implement vehicle search functionality that supports filtering by make, category, and price range.

---

## Prompt 5 – Inventory Management

Implement purchase and restock functionality where purchasing decreases inventory quantity and restocking increases inventory quantity while preventing invalid operations.

---

## Prompt 6 – Testing

Write Jest and Supertest test cases for authentication, vehicle CRUD operations, purchasing, restocking, authorization, and error handling.

---

## Prompt 7 – UI Development

Design a responsive and modern React + Tailwind CSS interface for both customer and administrator dashboards with clean navigation and intuitive user experience.

---

## Prompt 8 – Debugging

Help identify and resolve backend issues related to authentication, MongoDB integration, API responses, middleware, and failing test cases.

---

## Prompt 9 – Documentation

Generate a professional README.md including installation instructions, project features, API endpoints, testing information, AI usage disclosure, and project overview.

---

# AI Usage Statement

AI tools were used to assist with brainstorming, debugging, documentation, test improvements, UI suggestions, and explaining concepts.

All generated content was manually reviewed, modified, tested, and integrated before submission. Final implementation decisions and code verification were completed by the project author.
